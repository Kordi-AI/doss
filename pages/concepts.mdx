# Concepts

## The vault

Your vault is a plain folder (default `~/.doss`), synced across devices with git.

```text
~/.doss/
  self/          facts about you; the path IS the topic (self/profile/dietary.md)
  peers/         what other people shared with you
  notes/         markdown scratch; never shared
  policy.yaml    disclosure rules: who may see each topic as no/rough/full (syncs)
  devices/       synced device registry; active/unregistered YAML per device
  ledger/        the disclosure log, one append-only file per device
  local/         device-only settings (gitignored, never syncs)
    access.yaml  what your agent may DO with this device's folders, per group
  INSTRUCTION.md entry rules agents read first
  CONTENT.md     content maintenance rules
  DISCLOSURE.md  outbound disclosure and local access rules
```

A **fact** is a small markdown file with YAML frontmatter. Every `self/**/*.md`
fact needs `rough`, the owner-authored coarse value used for rough disclosure.
Git records time.

```markdown
<!-- self/profile/address.md -->
---
source: owner
status: active
confidence: high
rough: "Toronto"
---
Home address: 123 King St W, Toronto.
```

## Disclosure — `policy.yaml`

`policy.yaml` answers **who can SEE what**. It maps groups of verified people to
disclosure levels for topics under `self/`: `no`, `rough`, or `full`. This is your
portable preference, so it **syncs** across devices.

```yaml
groups:
  friends:  [kordi:pedro, kordi:qiancx]
  contacts: [kordi:jiaxin]
can-see:
  friends:
    profile/address: rough   # share only the file's rough value
    profile/dietary: full    # share the full Markdown body
    work: rough              # applies to everything under self/work/
  contacts:
    profile/address: rough
  # anything not listed: no disclosure
```

Topics are paths without the `self/` prefix. A topic may name a folder (`work`) or
a specific fact (`profile/address`). Folder rules inherit to facts below them, and
a more specific topic wins. **Default is deny.** A person in several groups gets
the highest granted level, ordered `no < rough < full`.

<Callout type="info">
  `rough:` is the only value an agent may share for a `rough` grant. The Markdown
  body after frontmatter is the full private fact. There is no `full:` field and
  no `no:` field inside a fact.
</Callout>

## Device access — `local/access.yaml`

`access.yaml` answers **who can make your agent DO what** with this device's folders —
reading a file to share, or carrying out a task someone delegates (e.g. "fix this bug").

It is a different concept from `policy.yaml`, so it lives in `local/` and **never syncs**:
`policy.yaml` is your portable preference; this is about one computer's files.

```yaml
grants:
  friends:
    ~/Projects/opensource: full    # read, edit, run
    ~/Documents/public: read       # may read/share, not modify
  contacts:
    ~/Projects/opensource: read
```

Three levels: `no` (don't touch it for them) · `read` (read/share files there) ·
`full` (read + edit + run). Default is `no`.

## Multi-device

Every device holds a full copy of the vault. `doss sync` (git underneath) keeps them
aligned; conflicts abort safely with both versions preserved. `devices/` syncs and
records which devices are `active` or `unregistered`; old records stay so ledger
files remain interpretable. `local/` never syncs. Ledger files do sync, but each
device appends to its own file under `ledger/` so devices do not fight over one log
file.

`doss init` registers the current device and prints the same list as `doss devices`.
`doss uninstall` marks the current device `unregistered`, pushes that state, and only
then deletes the local vault. `doss devices unregister <id>` marks another existing
device unregistered. This is bookkeeping, not security revocation; revoke GitHub,
SSH, or token access too if a device is lost.

## Identity

The requester's identity comes from the **platform's authenticated id** (like
`kordi:pedro`), never from what a message claims. No verified identity → treated as a
stranger → nothing. Groups are defined in `policy.yaml`.
